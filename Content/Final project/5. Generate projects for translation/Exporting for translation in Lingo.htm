Exporting for translation in Lingo

If your translation workflow is based inside your organization, or if your LSP uses MadCap Lingo, you can export and translate entirely within the Lingo environment — no need for separate files.

  1. Export your Flare project (to see this procedure, go to Exporting Flare Project section).

  2. Create a Lingo Project from Flare (to see this procedure, go to Creating Lingo Project section).

  3. Translate directly in Lingo or export a LIPRJZIP bundle to send to your LSP. To create a LIPRJZIP bundle:

    1. Select View > File List.

    2. Select the files to be included in the bundle.

    3. In the local toolbar click the Create Bundle button.

    4. In the Bundle Settings area, select the Lingo Project Bundle and check the project files to include with the bundle.

    5. Include Project Translation Memories, as needed.

    6. Include Project Termbases, as needed.

    7. Click OK. Statistics for the project will appear.

    8. Click Close. A subfolder (within your project folder) named "Exporting" opens. This subfolder holds the LIPRJZIP file. This file contains the files that your translator needs to work with.

  4. Send the LIPRJZIP file (via email, FTP, etc.) to the translator along with a style guide, if available.

  5. Once translation is completed and reviewed, synchronize the translated content back into Flare.

Key considerations

  • Seamless reintegration — no need for manual imports or file conversions.

  • Higher initial setup time but much smoother for large projects.

  • Recommended if you want full in-house control over localization.

  • Especially valuable when maintaining multiple language versions with frequent updates.
