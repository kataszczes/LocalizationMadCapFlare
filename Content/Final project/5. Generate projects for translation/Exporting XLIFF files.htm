Exporting XLIFF files

XLIFF (XML Localization Interchange File Format) is a standard for exchanging localization data. Although Flare does not generate XLIFF files directly, you can export your Flare content to MadCap Lingo, and from there, create XLIFF files for translation.

  1. Export your Flare project (to see this procedure, go to Exporting Flare Project section).

  2. Create a Lingo Project from Flare (to see this procedure, go to Creating Lingo Project section).

  3. Export an XLIFF bundle to send to your LSP:

    1. Select View → File List.

    2. Select the files to be included in the bundle.

    3. In the local toolbar click the Create Bundle button.

    4. In the Bundle Settings area, select the XLIFF Files Bundle and check the project files to include with the bundle.

    5. Include Project Translation Memories, as needed.

    6. Include Project Termbases, as needed.

    7. Click OK. Statistics for the project will appear.

    8. Click Close. A subfolder (within your project folder) named "Exporting" opens. This subfolder holds the ZIP file. This file contains the files that your translator needs to work with.

  4. Deliver XLIFF files to the LSP along with a style guide, if available.

  5. Once translation is completed and reviewed, import the XLIFF files back into the Lingo project.

  6. Export the Lingo project into a Flare project in the target language.

Key considerations

  • XLIFF handling is done only in MadCap Lingo, not Flare.

  • Lingo acts as the bridge between Flare and external translation tools.

  • This approach ensures that complex content (conditions, variables, snippets) is protected and preserved during translation.

  • It allows easy re-import of translations into Flare without losing structure.
